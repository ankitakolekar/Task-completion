Thinkbridge Assessment Program — Days 1 through 4

Building My First Web Page (HTML basics)

On Day 1 we learned basic HTML structure (doctype, html, head, body), headings, paragraphs and linking CSS.

Example

<!DOCTYPE html>
<html>
  <head>
    <title>My First Web Page</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    <p>This is my first webpage using HTML and CSS.</p>
  </body>
</html>

Lists & Semantic Markup

We explored ordered lists, unordered lists and nested lists — used for navigation, steps, and outlines.

Examples:
  • Unordered list: <ul><li>Item</li>...</ul>
  • Ordered list: <ol><li>Step 1</li>...</ol>
  • Nested lists: place a <ul> or <ol> inside an <li> for sub-items.

Images, Links & Accessibility

Added images with <img>, using src and descriptive alt text. Used anchor links and fragment identifiers.

Image example:

Always include meaningful alt text for accessibility and SEO.

Common Git & GitHub Commands

# configure identity
    git config --global user.name "Your Name"
    git config --global user.email you@example.com

    # clone, branch, commit & push
    git clone https://github.com/your-username/your-repo.git
    git checkout -b feature/short-description
    git add .
    git commit -m "Add feature: short description"
    git push -u origin feature/short-description

    # create PR with GitHub CLI (optional)
    gh auth login
    gh pr create --title "Short title" --body "Description" --base main

Pull Request / Repo Checklist

  • Use descriptive branch names (feature/, fix/, docs/)
  • Write a clear PR title and description; include screenshots if UI changes
  • Link related issue numbers (e.g., fixes #123)
  • Run tests & lint before requesting review
  • Add a short changelog or release note when appropriate

CI, Workflows & Pages

- Add a GitHub Actions workflow (.github/workflows/) for tests, linting, and deploy.
- Use GitHub Pages for static sites: Settings → Pages → deploy branch (or use actions deploy).
- Protect main branch: require PR reviews and passing checks.
